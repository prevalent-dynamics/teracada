Attention
All api functions MUST make sure that whenever they ERREXIT, they should properly call throwException(ERR), either just before goto ERREXIT (all), or in the ERREXIT label or both if it is better, as throwException will not overwrite previously set error. This step can only be skipped if it sure/known that throwException(ERR) has definitely been called by one of the previous function calls.
Note
For both tc_str and tc_char*, strings must be null terminated, before passing to the api functions
